# React File Upload Practice

A dedicated side project built with **Next.js** and **Tailwind CSS** to explore and implement best practices for file uploading in modern web applications. This repository serves as a playground for mastering various uploading techniques and UI/UX patterns.

## 🚀 Key Features

- **Strict File Validation**: Custom restrictions to only accept specific formats like `PNG`, `JPG`, `SVG`, and `GIF`.
- **Responsive Design**: Mobile-first approach using Tailwind CSS for a seamless experience across all devices.
- **Modular Components**: Cleanly separated logic for icons, inputs, and container components to ensure maintainability.
- **Multiple File Support**: Native support for handling multiple file selections.

## 🛠️ Tech Stack

- **Framework**: [Next.js 15](https://nextjs.org/)
- **Styling**: [Tailwind CSS](https://tailwindcss.com/)
- **Runtime**: [Bun](https://bun.sh/)
- **Icons**: Custom SVG icons and ReactLucide

## 📂 Project Structure

- `components/`: Pure UI and logic components for the upload interface.
- `app/`: Next.js App Router for layout and routing.

## 📖 What I Practiced

- Handling HTML5 File API.
- Styling hidden file inputs for custom "Click to upload" triggers.
- Configuring environment-specific file restrictions.
- Implementing accessible and user-friendly upload states.
